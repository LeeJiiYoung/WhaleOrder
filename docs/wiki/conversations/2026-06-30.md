## 가설 1 — 백엔드가 로그인 중 죽는다 (가장 의심) 🔴

`docker ps`에서 **RESTARTS가 늘어나거나 Up 시간이 짧으면** 이쪽입니다.

근거: [Dockerfile:22](Dockerfile#L22)에서 `-Xmx256m`인데, t2.micro(1GB)에 backend + postgres + redis가 같이 떠 있습니다. 로그인은 사용자가 테스트한 경로 중 **유일하게 BCrypt(CPU/메모리) + JWT 생성 + Redis 쓰기를 모두 타는 무거운 요청**([MemberService.java:73-83](src/main/java/com/whale/order/domain/member/service/MemberService.java#L73-L83))이라, 메모리 압박 시 JVM이 OOM-kill 되면 요청 중 연결이 끊겨 nginx가 502를 냅니다. `restart: unless-stopped`라 잠시 후 다시 떠서 "가끔 되는" 것처럼 보입니다.

확인: `docker logs whale-backend`에 `OutOfMemoryError` 또는 컨테이너 종료, 호스트에서 `dmesg | grep -i oom`.

## 가설 2 — Redis에 닿지 못해 로그인이 멈춘다

`docker logs`에 `RedisConnectionException` / Lettuce 타임아웃이 보이면 이쪽입니다. 로그인은 [RefreshTokenService.save()](src/main/java/com/whale/order/global/auth/RefreshTokenService.java#L25-L32)로 Redis에 쓰는데, 공개 조회 API는 Redis를 안 탑니다 — 그래서 "로그인만" 실패하는 증상과 맞습니다.
```bash
docker exec whale-redis redis-cli ping   # PONG 나와야 정상
docker network inspect whale-order_whale-net | grep -A3 whale-redis
```

## 가설 3 — nginx가 백엔드 옛 IP를 캐싱

[deploy.yml:52](.github/workflows/deploy.yml#L52)이 `--force-recreate backend frontend`로 백엔드를 새로 만드는데, [nginx.conf:14](frontend/nginx.conf#L14)의 `proxy_pass http://backend:8080;`은 **시작 시 DNS를 한 번만 해석**합니다. nginx가 백엔드보다 먼저 떠버리면 옛/없는 IP를 잡아 **모든 /api가 502**가 됩니다. 이 경우 로그인뿐 아니라 전부 502일 겁니다.

빠른 확인/임시 복구:
```bash
docker compose -f docker-compose.prod.yml restart frontend   # nginx 재시작 → 재해석
```
근본 해결은 nginx에 resolver + 변수 proxy_pass를 쓰는 것입니다.

1. **Hikari 커넥션 풀이 과대** — [application-prod.yaml:13-18](src/main/resources/application-prod.yaml#L13-L18): `maximum-pool-size: 30`, `minimum-idle: 10`. min-idle 10이면 **항상 10개의 Postgres 연결이 열려 있고**, Postgres는 연결당 프로세스(~5–10MB)를 띄웁니다 → 쓰지도 않는데 ~50–100MB 상시 점유.

## 코드 밖에서 — 가장 효과 큰 한 방: 스왑 추가

t2.micro의 OOM-kill을 막는 **가장 확실한 방법은 스왑 파일**입니다. 1GB RAM에 2GB 스왑을 붙이면 JVM이 순간 메모리 스파이크를 넘길 수 있어 502가 거의 사라집니다. EC2에서 한 번만:

```bash
sudo fallocate -l 2G /swapfile && sudo chmod 600 /swapfile
sudo mkswap /swapfile && sudo swapon /swapfile
echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab   # 재부팅 후에도 유지
free -h   # Swap 2.0Gi 확인
```
