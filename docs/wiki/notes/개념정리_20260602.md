## 1. Kafka DLQ (Dead Letter Queue) + Saga 보상 트랜잭션

### 개념

**DLQ**: 처리에 실패한 Kafka 메시지를 버리지 않고 별도 보관하는 큐.  
**Saga 보상**: 분산 환경에서 일부 단계가 실패했을 때 이미 성공한 단계를 되돌리는 로직.

```
DLQ  = "어디에" 실패를 모을지 (인프라)
Saga = "어떻게" 실패를 처리할지 (비즈니스 로직)
```

### 전체 흐름

```
주문 메시지 발행 (order-created)
      ↓
Consumer 처리 실패
      ↓
Kafka 자동 재시도 (1초 간격, 3회)
      ↓
그래도 실패 → order-created.DLT 로 이동
      ↓
DLT Consumer 수신
      ↓
Saga 보상 트랜잭션 실행 (재고 복구 → 결제 취소 → 주문 취소)
```

### 예외 분류

| 예외 종류 | 처리 방식 |
|---|---|
| DB 연결 끊김, 타임아웃 | 재시도 3회 → DLT |
| 잘못된 데이터, 없는 주문 ID | 바로 DLT (재시도 무의미) |

### 주요 소스

**`KafkaConfig.java`**
```java
// DLT 토픽 생성
@Bean
public NewTopic orderCreatedDltTopic() {
    return TopicBuilder.name("order-created.DLT").partitions(1).replicas(1).build();
}

// 재시도 3회 후 DLT 이동 설정
@Bean
public DefaultErrorHandler kafkaErrorHandler(KafkaTemplate<String, Object> kafkaTemplate) {
    DeadLetterPublishingRecoverer recoverer = new DeadLetterPublishingRecoverer(kafkaTemplate);
    DefaultErrorHandler handler = new DefaultErrorHandler(recoverer, new FixedBackOff(1000L, 3));
    handler.addNotRetryableExceptions(IllegalArgumentException.class, IllegalStateException.class);
    return handler;
}
```

**`OrderKafkaConsumer.java`**
```java
// 일반 Consumer: 실패 시 예외 throw → Kafka 재시도 → DLT 이동
@KafkaListener(topics = "order-created", groupId = "whale-order")
public void consume(Long orderId) { ... throw e; }

// DLT Consumer: 보상 트랜잭션 실행
@KafkaListener(topics = "order-created.DLT", groupId = "whale-order-dlt")
public void consumeDlt(Long orderId, @Header(KafkaHeaders.EXCEPTION_MESSAGE) String exceptionMessage) {
    orderProcessingService.compensate(orderId);
}
```

**`OrderProcessingService.java`**
```java
// 보상 트랜잭션 진입점
public void compensate(Long orderId) {
    cancelOrder(orderId);
}

// 이미 취소된 주문이면 스킵 (가드)
protected void cancelOrder(Long orderId) {
    orderRepository.findById(orderId).ifPresent(order -> {
        if (order.getStatus() == OrderStatus.CANCELLED) return; // 중복 실행 방지
        order.cancel();
        // 결제 취소 (Saga 보상)
        payment.cancel("재고 부족으로 인한 자동 환불");
    });
}
```

### 재고 복구 실패 처리

재고 복구 실패 시 로그만 찍고 넘어가면 재고 불일치가 발생한다.  
→ DB에 기록 + 관리자 SSE 알림으로 수동 보정 유도.

- `StockRestoreFailure.java` — 실패 기록 엔티티

- `/admin/stock-restore-failures` — 관리자 화면 (React)

## 2. Refresh Token Rotation (RTR)

### 개념

Access Token(30분)이 만료됐을 때 사용자가 로그아웃 없이 자동으로 갱신되는 구조.  
**RTR**: Access Token 갱신 시 Refresh Token도 함께 교체 → 탈취된 Refresh Token 재사용 차단.

### 두 토큰의 역할

| | Access Token | Refresh Token |
|---|---|---|
| 수명 | 30분 | 7일 |
| 용도 | 실제 API 호출 | Access Token 재발급에만 사용 |
| 저장 위치 | localStorage | localStorage + Redis |
| 짧게 쓰는 이유 | 탈취돼도 30분만 피해 | 편의성 (7일 유지) |

### Refresh Token을 Redis에 저장하는 이유

- TTL 자동 관리 — 7일 후 Redis가 알아서 삭제 (스케줄러 불필요)
- 로그아웃 시 즉시 무효화 가능 (JWT는 자체 무효화 불가)

### 전체 흐름

```
로그인
  ↓
Access Token + Refresh Token 발급
Refresh Token → Redis 저장 (key: refresh:{memberId}, TTL 7일)

API 요청 중 Access Token 만료 → 401
  ↓
프론트 인터셉터가 POST /api/auth/refresh 자동 호출
  ↓
백엔드: Redis 검증 → 기존 Refresh Token 폐기 → 새 토큰 쌍 발급
  ↓
새 Access Token으로 원래 요청 재시도 (사용자는 모름)

7일 후 Refresh Token도 만료
  ↓
로그아웃
```

### RTR 보안 포인트

```
탈취된 Refresh Token으로 갱신 시도
  ↓
Redis에 저장된 토큰과 불일치 감지
  ↓
저장된 토큰도 즉시 삭제 (재사용 완전 차단)
  ↓
정상 사용자도 로그아웃 → 재로그인 유도
```

### 주요 소스

**`RefreshTokenService.java`**
```java
// Redis 저장 (TTL 7일)
public void save(Long memberId, String refreshToken) {
    redisTemplate.opsForValue().set("refresh:" + memberId, refreshToken, refreshTokenExpiry, TimeUnit.MILLISECONDS);
}
public String get(Long memberId) { ... }
public void delete(Long memberId) { ... }
```

**`MemberService.java`**
```java
// RTR: 검증 → 폐기 → 새 토큰 발급
public LoginResponse refresh(String refreshToken) {
    Long memberId = jwtProvider.getMemberId(refreshToken);
    String stored = refreshTokenService.get(memberId);
    if (!refreshToken.equals(stored)) {
        refreshTokenService.delete(memberId); // 탈취 감지 시 즉시 삭제
        throw new IllegalArgumentException("리프레시 토큰이 일치하지 않습니다");
    }
    refreshTokenService.delete(memberId); // 기존 폐기
    return issueTokens(member);           // 새 토큰 쌍 발급
}

// 로그인 시 Redis 저장
private LoginResponse issueTokens(Member member) {
    String refreshToken = jwtProvider.generateRefreshToken(member.getMemberId());
    refreshTokenService.save(member.getMemberId(), refreshToken); // Redis 저장
    ...
}
```

**`AuthController.java`**
```
POST /api/auth/refresh  — Access + Refresh Token 동시 갱신
POST /api/auth/logout   — Redis에서 Refresh Token 즉시 삭제
```
